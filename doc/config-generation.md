# DNS Builder 配置生成与挂载机制

本文档详细说明 DNS Builder 如何为不同的 DNS 软件生成配置文件并进行挂载。

## 1. 核心概念

### 1.1 GENERATED_ZONES_FILENAME

`GENERATED_ZONES_FILENAME = "generated_zones.conf"` 是由 behavior 自动生成的配置文件，用于存放所有 behavior 产生的配置行。

**生成流程** (`service.py:591-608`):

1. 收集所有 behavior 生成的配置行（按 section 分类）
2. 写入到 `temp:/services/{service_name}/generated_zones.conf`
3. 挂载到容器路径 `/usr/local/etc/zones/generated_zones.conf`

### 1.2 BehaviorArtifact

`BehaviorArtifact` 定义了 behavior 生成的输出结构：

```python
class BehaviorArtifact(BaseModel):
    config_line: str                              # 配置行内容
    new_volume: Optional[VolumeArtifact] = None  # 需要生成的文件（如 zone 文件）
    section: constants.BehaviorSection = TOPLEVEL # 配置所属的 section
    new_records: Optional[List[RR]] = None       # Master behavior 生成的 DNS 记录
```

### 1.3 DNS_SOFTWARE_BLOCKS

定义各软件支持的配置块：

| 软件 | 支持的块 |
|------|---------|
| **bind** | global, acl, controls, options, logging, zone, view, key, server, trusted-keys, managed-keys, statistics-channels |
| **unbound** | global, server, remote-control, stub-zone, forward-zone, auth-zone, view, python, dynlib |
| **pdns_recursor** | global |
| **knot_resolver** | global |

## 2. Behavior 配置生成

### 2.1 配置生成对照表

| 软件类型 | Forward | Stub | Master | Hint |
|---------|---------|------|--------|------|
| **BIND** | `zone "x" { type forward; forwarders {...}; };` | `zone "x" { type stub; masters {...}; };` | `zone "x" { type master; file "..."; };` | 生成 root.hints 文件 + zone 配置 |
| **Unbound** | `forward-zone:\n\tname: "x"\n\tforward-addr: ...` | `stub-zone:\n\tname: "x"\n\tstub-addr: ...` | `auth-zone:\n\tname: "x"\n\tzonefile: "..."` | `root-hints: "..."` (server section) |
| **PDNS Recursor** | `forward-zones+=zone=ip;ip` | `forward-zones+=zone=ip;ip` | `auth-zones+=zone=file` | `hint-file=...` |
| **Knot Resolver** | `policy.add(policy.suffix(policy.FORWARD(...)))` | `policy.add(policy.suffix(policy.STUB(...)))` | N/A | `hints.root({...})` |

### 2.2 各软件 Behavior 详细配置

#### BIND (bind)

**Forward Behavior**:
```bind
zone "example.com" {
    type forward;
    forwarders { 192.168.1.1; 192.168.1.2; };
};
```

**Stub Behavior**:
```bind
zone "example.com" {
    type stub;
    masters { 192.168.1.1; 192.168.1.2; };
};
```

**Master Behavior**:
```bind
zone "example.com" {
    type master;
    file "/usr/local/etc/zones/db.example.com";
};
```

**Hint Behavior**:
```bind
zone "." {
    type hint;
    file "/usr/local/etc/zones/gen_service_root.hints";
};
```

#### Unbound

**Forward Behavior**:
```yaml
forward-zone:
    name: "example.com"
    forward-addr: 192.168.1.1
    forward-addr: 192.168.1.2
```

**Stub Behavior**:
```yaml
stub-zone:
    name: "example.com"
    stub-addr: 192.168.1.1
    stub-addr: 192.168.1.2
```

**Master Behavior**:
```yaml
auth-zone:
    name: "example.com"
    zonefile: "/usr/local/etc/unbound/zones/db.example.com"
```

**Hint Behavior** (server section):
```yaml
server:
    root-hints: "/usr/local/etc/unbound/zones/gen_service_root.hints"
```

#### PowerDNS Recursor

**Forward/Stub Behavior**:
```ini
forward-zones+=example.com=192.168.1.1;192.168.1.2
```

**Master Behavior**:
```ini
auth-zones+=example.com=/usr/local/etc/zones/db.example.com
```

**Hint Behavior**:
```ini
hint-file=/usr/local/etc/zones/gen_service_root.hints
```

#### Knot Resolver

**Forward Behavior**:
```lua
policy.add(policy.suffix(policy.FORWARD({'192.168.1.1', '192.168.1.2'}), {todname('example.com')}))
```

**Stub Behavior**:
```lua
policy.add(policy.suffix(policy.STUB({'192.168.1.1', '192.168.1.2'}), {todname('example.com')}))
```

**Hint Behavior**:
```lua
modules.load('hints')
hints.root({['A.ROOT-SERVERS.NET.'] = {'192.168.1.1'}, ['B.ROOT-SERVERS.NET.'] = {'192.168.1.2'}})
```

## 3. Includer 机制

当用户通过 `volumes` 挂载多个 `.conf` 配置文件时，Includer 负责在主配置文件中添加 include 指令。

### 3.1 配置文件命名约定

通过文件后缀解析所属块：

| 文件名 | 解析结果 | 说明 |
|--------|---------|------|
| `named.conf` | global | 主配置文件 |
| `named.conf.options` | options | options 块配置 |
| `named.conf.logging` | logging | logging 块配置 |
| `named.conf.zone` | zone | zone 块配置 |

### 3.2 各软件 Includer 实现

#### BindIncluder

```python
class BindIncluder(Includer):
    _tmpl = '\n# Auto Generated by DNS-Builder\ninclude "{config_line}";\n'
    _blk = '\n# Auto Generated by DNS-Builder\n{block} {{{lines}}};\n'
```

**特点**:
- 支持分块配置
- 自动在对应块内插入 include 指令
- 如果块不存在，自动创建块结构

**生成示例**:
```bind
# Auto Generated by DNS-Builder
options {
    # Auto Generated by DNS-Builder
    include "/usr/local/etc/options/custom.conf";
};
```

#### UnboundIncluder

```python
class UnboundIncluder(Includer):
    _tmpl = '\n# Auto Generated by DNS-Builder\ninclude: "{config_line}"\n'
```

**特点**:
- 支持分块配置
- 为块配置自动添加块标识符

**生成示例**:
```yaml
# Auto-Include options block by DNS Builder
options:
    # Auto Generated by DNS-Builder
    include: "/usr/local/etc/options/custom.conf"
```

#### PdnsRecursorIncluder

```python
class PdnsRecursorIncluder(Includer):
    _tmpl = '\n# include {config_line}'
    _write = "\n# Auto Generated by DNS Builder\ninclude-dir={include_dir}\n"
```

**特点**:
- 将配置文件移动到统一的 includes 目录
- 只在主配置中添加 `include-dir` 指令
- 返回修改后的挂载路径

**生成示例**:
```ini
# Auto Generated by DNS Builder
include-dir=/usr/local/etc/includes
# include /path/to/file.conf -> /usr/local/etc/includes/file.conf
```

#### KnotResolverIncluder

```python
class KnotResolverIncluder(Includer):
    _tmpl = "\n-- Auto Generated by DNS Builder\ndofile('{config_line}')\n"
```

**特点**:
- 使用 Lua 语法的 `dofile()` 函数
- 只支持全局配置

**生成示例**:
```lua
-- Auto Generated by DNS Builder
dofile('/usr/local/etc/custom.lua')
```

## 4. 整体处理流程

### 4.1 流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                    ServiceHandler.generate_all()                │
└─────────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐   ┌─────────────────┐   ┌───────────────────┐
│ _process_files│   │_process_behavior│   │ _process_volumes  │
│   (files)     │   │   (behavior)    │   │   (volumes)       │
└───────────────┘   └─────────────────┘   └───────────────────┘
        │                     │                     │
        │           ┌─────────┴─────────┐           │
        │           ▼                   ▼           │
        │   ┌───────────────┐   ┌───────────────┐   │
        │   │standard_      │   │master_        │   │
        │   │artifacts      │   │artifacts      │   │
        │   └───────────────┘   └───────────────┘   │
        │           │                   │           │
        │           ▼                   ▼           │
        │   ┌─────────────────────────────────┐     │
        │   │  generated_zones.conf           │     │
        │   │  (所有 config_line 写入此文件)    │     │
        │   └─────────────────────────────────┘     │
        │                     │                     │
        └─────────────────────┼─────────────────────┘
                              ▼
                    ┌─────────────────┐
                    │ Includer.include│
                    │ (处理额外配置)   │
                    └─────────────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │ docker-compose  │
                    │   volumes:      │
                    └─────────────────┘
```

### 4.2 关键步骤说明

#### Step 1: _process_files

处理用户通过 `files` 字段定义的内联文件内容：

```yaml
services:
  my-dns:
    files:
      /etc/custom.conf: |
        # custom configuration
        option value
```

生成临时文件并添加到 volumes。

#### Step 2: _process_behavior

处理 `behavior` 字段：

1. 解析每行 behavior 字符串
2. 根据 software 类型创建对应的 Behavior 实例
3. 调用 `generate()` 方法生成 BehaviorArtifact
4. 收集所有配置行到 generated_zones.conf
5. 生成并挂载 zone 文件（如 Master/Hint behavior）

#### Step 3: _process_volumes

处理用户定义的 `volumes`：

1. 解析每个 volume 的源路径和目标路径
2. 根据 `.conf` 后缀判断配置块
3. 如果需要复制，将文件复制到 contents 目录
4. 调用 Includer.include() 在主配置中添加 include 指令

## 5. 配置文件挂载路径

### 5.1 标准挂载路径

| 软件 | 主配置路径 | Zone 文件路径 | Include 目录 |
|------|-----------|--------------|-------------|
| BIND | `/usr/local/etc/named.conf` | `/usr/local/etc/zones/` | 主配置内 |
| Unbound | `/usr/local/etc/unbound/unbound.conf` | `/usr/local/etc/unbound/zones/` | 主配置内 |
| PDNS Recursor | `/usr/local/etc/recursor.conf` | `/usr/local/etc/zones/` | `/usr/local/etc/includes/` |
| Knot Resolver | `/etc/knot-resolver/kresd.conf` | `/etc/knot-resolver/` | 主配置内 |

### 5.2 generated_zones.conf 挂载

所有软件统一挂载到：
```
temp:/services/{service_name}/generated_zones.conf -> /usr/local/etc/zones/generated_zones.conf
```

## 6. 使用示例

### 6.1 BIND 示例

```yaml
images:
  bind-image:
    software: bind
    version: "9.18"

builds:
  bind-server:
    image: bind-image
    behavior: |
      Master example.com www A 192.168.1.10
      Forward forward.example.com bind-server
      Hint . root-server
    volumes:
      - ./configs/named.options.conf:/usr/local/etc/named.options.conf
```

**生成结果**:

1. `generated_zones.conf`:
```bind
zone "example.com" { type master; file "/usr/local/etc/zones/db.example.com"; };
zone "forward.example.com" { type forward; forwarders { 192.168.1.1; }; };
zone "." { type hint; file "/usr/local/etc/zones/gen_bind_server_root.hints"; };
```

2. `named.conf` 自动添加:
```bind
options {
    include "/usr/local/etc/named.options.conf";
};
include "/usr/local/etc/zones/generated_zones.conf";
```

### 6.2 Unbound 示例

```yaml
images:
  unbound-image:
    software: unbound
    version: "1.17"

builds:
  unbound-server:
    image: unbound-image
    behavior: |
      Master example.com www A 192.168.1.10
      Forward forward.example.com unbound-server
    volumes:
      - ./configs/server.conf:/usr/local/etc/unbound/server.conf
```

**生成结果**:

1. `generated_zones.conf`:
```yaml
auth-zone:
    name: "example.com"
    zonefile: "/usr/local/etc/unbound/zones/db.example.com"

forward-zone:
    name: "forward.example.com"
    forward-addr: 192.168.1.1
```

2. `unbound.conf` 自动添加:
```yaml
server:
    include: "/usr/local/etc/unbound/server.conf"
include: "/usr/local/etc/zones/generated_zones.conf"
```

## 7. 扩展机制

### 7.1 添加新软件支持

1. **创建 Behavior 类** (在 `bases/behaviors.py` 或插件中):

```python
class MyDNSForwardBehavior(Behavior):
    def generate(self, service_name: str, build_context: BuildContext) -> BehaviorArtifact:
        # 生成配置行
        config_line = f"forward-zone: {self.zone} -> {self.targets[0]}"
        return BehaviorArtifact(config_line=config_line)
```

2. **创建 Includer 类**:

```python
class MyDNSIncluder(Includer):
    _tmpl = '\n# Auto Generated\ninclude "{config_line}"\n'

    def include(self, pair: Pair):
        # 添加 include 逻辑
        pass

    def contain(self):
        # 处理块配置
        pass
```

3. **注册到 DNS_SOFTWARE_BLOCKS** (如需要块支持):

```python
DNS_SOFTWARE_BLOCKS["mydns"] = {"global", "server", "zone"}
```

### 7.2 插件开发

参考 `dnsb_axdns` 插件示例，通过插件系统注册自定义实现。
